## 1. 개요

### 1.1 제품 개요

**EduForum**은 대학교 및 고등교육기관을 위한 **액티브 러닝 중심 온라인 세미나 플랫폼**입니다. 미네르바 대학교의 Forum 시스템에서 영감을 받아, 단순한 화상 회의를 넘어 **실시간 참여도 추적, 자동 평가, 교육학적 상호작용 도구**를 통합한 차세대 교육 플랫폼을 제공합니다.

### 1.2 배경

전통적인 화상 회의 도구(Zoom, Google Meet, MS Teams)는:
- **수동적 학습**: 대형 강의 방식의 일방향 전달
- **참여도 파악 어려움**: 누가 얼마나 기여하는지 알 수 없음
- **교육학적 설계 부재**: 범용 도구로 교육 전용 기능 부족
- **피드백 지연**: 수업 후에야 학생 이해도 파악

EduForum은 이러한 문제를 해결하여 **학생 중심의 능동적 학습 환경**을 조성합니다.

### 1.3 제품 범위

#### In Scope (포함)
- 실시간 화상 세미나 (최대 50명)
- 소크라틱 토론, 투표, 퀴즈, 분반 활동
- 실시간 참여도 추적 및 대시보드
- 자동 평가 및 즉각 피드백
- 코스 및 세션 관리
- 녹화 및 다시보기
- 기본 LMS 기능 (과제, 성적)

#### Out of Scope (제외 - 향후 고려)
- 비동기 강의 콘텐츠 제작 도구 (이미 존재하는 도구 활용)
- 전체 대학 행정 시스템 (학사 관리 등)
- 대규모 웨비나 (500명 이상)
- 모바일 네이티브 앱 (v1에서는 반응형 웹)

## 2. 제품 비전 및 목표

### 2.1 비전 (Vision)

> "모든 온라인 수업을 능동적 학습 경험으로 전환하여, 학생들이 비판적으로 사고하고 효과적으로 협업하는 능력을 개발할 수 있는 교육 환경을 제공한다."

### 2.2 미션 (Mission)

1. **학생 참여 극대화**: 모든 학생이 능동적으로 참여하는 수업
2. **데이터 기반 교육**: 실시간 분석으로 교수가 즉시 대응
3. **자동화된 피드백**: AI를 활용한 즉각적인 학습 피드백
4. **교육 접근성 향상**: 지리적 제약 없이 고품질 교육 제공

### 2.3 제품 목표

| 목표 | 설명 | 측정 지표 |
|------|------|-----------|
| **학생 참여도 증가** | 전통적 강의 대비 2배 이상 참여 | 발언율, 투표 참여율 |
| **교수 만족도 향상** | 수업 운영 편의성 및 효과성 | NPS 점수 70+ |
| **학습 성과 개선** | 능동적 학습을 통한 이해도 향상 | 퀴즈 정답률, 학기말 성적 |
| **시장 점유율 확보** | 출시 1년 내 국내 대학 20곳 도입 | 고객 수 |
| **확장 가능성** | 동시 세션 1000개 지원 | 시스템 안정성 |

### 2.4 핵심 가치 제안 (Value Proposition)

#### 교수진에게
- ✅ **실시간 참여도 시각화**: 누가 뒤처지는지 즉시 파악
- ✅ **자동 평가 절감**: AI 채점으로 시간 절약
- ✅ **교육학적 도구**: 소크라틱 토론, 분반 활동 쉽게 운영
- ✅ **데이터 기반 개선**: 학기 전체 학습 패턴 분석

#### 학생들에게
- ✅ **능동적 참여**: 수동적 청취가 아닌 토론 중심
- ✅ **즉각적 피드백**: 이해도를 실시간으로 확인
- ✅ **공정한 평가**: 참여도가 정량적으로 측정
- ✅ **협업 기회**: 소그룹 활동으로 동료 학습

#### 교육기관에게
- ✅ **교육 품질 향상**: 검증된 액티브 러닝 방법론
- ✅ **데이터 인사이트**: 기관 전체 학습 분석
- ✅ **비용 효율**: 기존 도구 대비 교육 전용 기능
- ✅ **브랜드 차별화**: 혁신적 교육 기술 도입

## 3. 문제 정의

### 3.1 현재의 문제점

#### 문제 1: 수동적 학습 환경
- **현상**: 온라인 수업에서 학생들이 카메라를 끄고 수동적으로 참여
- **영향**: 학습 효과 저하, 중도 이탈률 증가
- **근거**: 연구에 따르면 수동적 청취는 장기 기억 형성에 비효율적

#### 문제 2: 참여도 파악 불가
- **현상**: 교수가 20-30명 학생의 참여도를 실시간으로 파악 불가
- **영향**: 뒤처지는 학생을 조기에 발견하지 못함
- **근거**: Zoom 등은 기본적인 발언 시간만 제공

#### 문제 3: 피드백 지연
- **현상**: 퀴즈나 과제를 수동으로 채점하여 피드백이 늦음
- **영향**: 학생들이 학습 공백을 장기간 방치
- **근거**: 연구에 따르면 즉각적 피드백이 학습 효과를 30% 이상 향상

#### 문제 4: 토론 관리 어려움
- **현상**: 온라인에서 소크라틱 토론이나 분반 활동 운영이 복잡
- **영향**: 교수가 액티브 러닝 방법론 포기
- **근거**: 교수 설문 조사 결과 80%가 "온라인 토론 어려움" 호소

### 3.2 해결 방안

EduForum은 다음과 같이 문제를 해결합니다:

| 문제 | EduForum 솔루션 | 기대 효과 |
|------|-----------------|-----------|
| 수동적 학습 | 투표, 퀴즈, 토론 강제 참여 구조 | 참여율 50% → 90% |
| 참여도 파악 불가 | 실시간 대시보드 (TalkTime™) | 즉시 개입 가능 |
| 피드백 지연 | AI 자동 채점 및 즉각 피드백 | 피드백 시간 2일 → 즉시 |
| 토론 관리 어려움 | 분반, 투표, 손들기 원클릭 운영 | 교수 준비 시간 50% 절감 |

## 5. 주요 기능 요구사항

### 5.1 기능 분류

기능을 **5대 핵심 영역**으로 분류:
1. **실시간 세미나 (Live Session)**
2. **액티브 러닝 도구 (Active Learning Tools)**
3. **평가 및 피드백 (Assessment & Feedback)**
4. **학습 분석 (Learning Analytics)**
5. **코스 관리 (Course Management)**

---

### 5.2 기능 영역 1: 실시간 세미나

#### F1.1 화상 회의 (Video Conferencing)
- **설명**: 최대 50명이 참여하는 실시간 화상 세미나
- **세부 기능**:
  - HD 비디오/오디오 (720p+)
  - 개별 카메라/마이크 제어
  - 레이아웃 모드: 발표자 중심, 그리드, 스포트라이트
  - 대역폭 자동 조정 (네트워크 상태 기반)
- **기술 요구사항**:
  - WebRTC 기반 SFU 아키텍처
  - 지연시간 < 300ms
- **우선순위**: **Must Have (P0)**

#### F1.2 화면 공유 (Screen Sharing)
- **설명**: 발표자가 전체 화면 또는 특정 창 공유
- **세부 기능**:
  - 전체 화면 / 애플리케이션 창 / 브라우저 탭 선택
  - 화면 공유 중 주석 도구
  - 원격 제어 권한 부여 (선택)
- **우선순위**: **Must Have (P0)**

#### F1.3 실시간 채팅 (Live Chat)
- **설명**: 세션 중 텍스트 기반 의사소통
- **세부 기능**:
  - 전체 채팅 / 개인 메시지
  - 파일 공유 (이미지, 문서)
  - 이모지 반응
  - 채팅 저장 및 다운로드
- **우선순위**: **Must Have (P0)**

#### F1.4 손들기 및 반응 (Raise Hand & Reactions)
- **설명**: 비언어적 신호로 참여 표현
- **세부 기능**:
  - 손들기 버튼 (큐에 자동 추가)
  - 빠른 반응: 👍 👏 ❓ 등
  - 교수가 손든 학생 목록 확인
- **우선순위**: **Must Have (P0)**

#### F1.5 세션 녹화 (Session Recording)
- **설명**: 세션 전체를 자동 녹화
- **세부 기능**:
  - 원클릭 녹화 시작/종료
  - 비디오, 오디오, 화면 공유, 채팅 모두 녹화
  - 클라우드 저장 (자동 인코딩)
  - 녹화본 다시보기 인터페이스
  - 자막 자동 생성 (음성인식)
- **우선순위**: **Should Have (P1)**

---

### 5.3 기능 영역 2: 액티브 러닝 도구

#### F2.1 실시간 투표 (Live Polling)
- **설명**: 수업 중 즉석 질문 및 투표
- **세부 기능**:
  - 질문 유형: 객관식, 다중 선택, 주관식, O/X
  - 익명 vs 기명 선택
  - 결과 실시간 시각화 (막대 그래프, 파이 차트)
  - 투표 후 정답 및 해설 표시
  - 투표 기록 저장 (참여 추적)
- **사용 시나리오**:
  - 사전 학습 확인 (수업 시작 시)
  - 개념 이해도 체크 (중간 중간)
  - 의견 조사 (토론 주제 선정)
- **우선순위**: **Must Have (P0)**

#### F2.2 퀴즈 시스템 (Quiz System)
- **설명**: 사전 학습 확인 및 평가용 퀴즈
- **세부 기능**:
  - 문제 은행 (Question Bank) 관리
  - 문제 유형: 객관식, 주관식, 참/거짓, 코딩 문제
  - 시간 제한 설정
  - 자동 채점 (객관식)
  - AI 유사도 채점 (주관식)
  - 즉각적 피드백 (정답/해설)
  - 퀴즈 결과 분석 (문항별 정답률)
- **우선순위**: **Must Have (P0)**

#### F2.3 분반 (Breakout Rooms)
- **설명**: 소그룹 토론 및 협업 활동
- **세부 기능**:

### 6.2 확장성 (Scalability)

- **수평 확장**: Kubernetes Auto-scaling으로 트래픽 증가 시 자동 확장
- **데이터베이스**: Read Replica로 읽기 부하 분산
- **미디어 서버**: 지역별 분산 배치 (Multi-Region)
- **CDN**: 정적 파일 및 녹화 영상 CDN 제공

### 6.3 가용성 (Availability)

- **목표 Uptime**: 99.9% (월 43분 다운타임 허용)

- **백업**: 데이터베이스 일일 백업, 7일 보관
- **재해 복구**: Multi-AZ 배포, 복구 시간 < 1시간

- **데이터 암호화**:
  - 전송 중: TLS 1.3
  - 저장 시: AES-256

- **권한**: RBAC (Role-Based Access Control)
- **개인정보 보호**:
  - GDPR 준수 (유럽)
  - FERPA 준수 (미국)
  - 학생 동의 없이 데이터 외부 공유 금지
- **보안 감사**: 분기별 취약점 스캔

### 6.5 사용성 (Usability)

- **직관적 UI**: 3클릭 이내 모든 주요 기능 접근
- **반응형 디자인**: 데스크탑, 태블릿, 모바일 지원

## 7. 사용자 스토리

### 7.1 Epic 1: 실시간 세미나 진행

#### US-1.1: 세미나 시작
**As** 교수
**I want to** 예정된 세미나를 원클릭으로 시작하고 학생들이 자동으로 입장하도록
**So that** 수업 시작이 빠르고 원활하다

**Acceptance Criteria**:
- [ ] 스케줄된 세미나가 "시작" 버튼 표시
- [ ] 시작 버튼 클릭 시 3초 이내 세미나 룸 생성
- [ ] 학생들에게 실시간 알림 (앱, 이메일)
- [ ] 학생들이 링크 클릭 시 즉시 입장 (별도 로그인 불필요)

#### US-1.2: 비디오 참여
**As** 학생
**I want to** 비디오와 오디오를 켜고 끄며 참여
**So that** 네트워크 상황에 맞게 조정할 수 있다

**Acceptance Criteria**:
- [ ] 카메라/마이크 on/off 버튼 명확
- [ ] 네트워크 속도가 느릴 때 자동으로 비디오 품질 하향 조정
- [ ] 다른 참가자들이 내 상태를 실시간으로 확인 가능

### 7.2 Epic 2: 액티브 러닝 활동

#### US-2.1: 실시간 투표
**As** 교수
**I want to** 수업 중 즉석 질문을 던지고 학생들의 응답을 실시간으로 확인
**So that** 학생들의 이해도를 즉시 파악하고 수업을 조정할 수 있다

**Acceptance Criteria**:
- [ ] 투표 생성 시간 < 30초
- [ ] 학생들에게 즉시 투표 팝업 표시
- [ ] 응답률 실시간 업데이트 (50% → 80% → 100%)
- [ ] 결과를 차트로 시각화 (막대, 파이)
- [ ] 투표 종료 후 정답 및 해설 표시

#### US-2.2: 분반 토론
**As** 교수
**I want to** 학생들을 4-5명씩 소그룹으로 나누어 토론시키고 각 그룹을 순회하며 모니터링
**So that** 모든 학생이 능동적으로 참여하고 소그룹 토론이 효과적으로 진행된다

**Acceptance Criteria**:
- [ ] 그룹 수와 배정 방식 선택 (자동/수동)
- [ ] 10초 이내 모든 학생이 분반으로 이동
- [ ] 각 분반은 독립된 비디오/오디오/채팅
- [ ] 교수가 그룹 간 자유 이동 (클릭 한 번)
- [ ] 타이머 설정 및 전체 복귀 알림
- [ ] 분반 활동 시간 자동 추적

## 11. 제약사항 및 가정

### 11.1 제약사항

#### 기술적 제약
- **브라우저 의존성**: WebRTC 지원 브라우저 필수 (IE 불가)
- **네트워크 요구사항**: 안정적인 인터넷 연결 필요 (최소 1 Mbps)
- **동시 참여자**: v1에서는 세션당 최대 50명
- **녹화 용량**: 클라우드 스토리지 비용 증가 (고객 부담)

#### 비즈니스 제약
- **예산**: 초기 개발 예산 제한 (인력 5-10명)
- **시장**: 한국 대학 시장 우선, 해외는 v2 이후
- **경쟁**: Zoom, MS Teams 등 대형 업체와 경쟁
- **규제**: GDPR, FERPA 등 국가별 규정 준수 필요

#### 리소스 제약
- **개발 인력**: 5-10명 (백엔드 3, 프론트엔드 2, DevOps 1, QA 1, 디자인 1)
- **개발 기간**: MVP까지 4개월, v1.0까지 9개월
- **인프라 비용**: 초기 월 $5,000-$10,000 (AWS/Azure)

### 11.2 가정 (Assumptions)

#### 사용자 행동 가정
- 교수들은 새로운 도구를 배우는 데 최대 30분 투자
- 학생들은 모바일보다 PC/노트북으로 주로 접속
- 평균 세션 길이는 90분
- 세션당 평균 20명 참여

#### 시장 가정
- 코로나19 이후 온라인/하이브리드 교육 지속 증가
- 대학들은 교육 품질 향상을 위해 전용 도구 투자 의향 있음
- 교수들은 참여도 추적 및 자동 채점에 높은 가치 부여

#### 기술 가정
- WebRTC 기술은 충분히 성숙하여 안정적 서비스 가능
- 오픈소스 (Jitsi, Django 등) 활용으로 빠른 개발 가능
- AI 자동 채점 정확도는 교수 검토 후 80%+ 달성 가능

### 12.2 경쟁 우위 (Competitive Advantage)

EduForum의 **3대 차별화 요소**:

1. **실시간 참여도 추적**
   - Zoom 등은 단순 발언 시간만 제공
   - EduForum은 발언, 채팅, 투표, 퀴즈 모두 통합 추적
   - 교수가 즉시 개입 가능 (뒤처진 학생 질문 유도)

2. **AI 기반 자동 평가**
   - 주관식 에세이 NLP 채점
   - 코딩 문제 자동 실행
   - 즉각적 피드백 (학습 효과 30% 향상)

3. **교육학적 설계**
   - 단순 화상회의가 아닌 액티브 러닝 전용
   - 소크라틱 토론, 분반, 투표 등 원클릭 운영
   - 인지과학 연구 기반 마이크로 모듈 구조

### 12.3 진입 장벽 (Entry Barriers)

#### 우리의 장벽
- 교수/학생 행동 변화 유도 (기존 도구에 익숙)
- Zoom 등 대기업의 브랜드 인지도
- 대학 의사결정 느림 (구매 주기 길음)

#### 경쟁사의 장벽
- 교육 전용 기능 개발에는 전문성 필요
- 참여도 추적 및 AI 채점은 복잡한 기술
- 교육학적 이해도 필요 (기술만으로 부족)

## 13. 위험 요소 및 대응

### 13.1 기술 위험

#### Risk 1: WebRTC 안정성
- **위험**: 네트워크 환경에 따라 연결 불안정
- **영향**: 높음 - 핵심 기능 장애
- **확률**: 중간
- **대응**:
  - TURN 서버 충분히 확보
  - 자동 재연결 로직
  - 네트워크 진단 도구 제공

#### Risk 2: 확장성 문제
- **위험**: 동시 사용자 증가 시 성능 저하
- **영향**: 높음 - 서비스 품질 저하
- **확률**: 중간
- **대응**:
  - Kubernetes Auto-scaling
  - 사전 부하 테스트
  - 점진적 확장 (한 학기 단위)

#### Risk 3: AI 채점 정확도
- **위험**: 주관식 채점 오류로 학생 불만
- **영향**: 중간 - 신뢰도 저하
- **확률**: 높음 (초기)
- **대응**:
  - 교수 최종 검토 필수
  - 학습 데이터 지속 개선
  - 신뢰도 점수 표시 (80% 이상만 자동)

### 13.2 비즈니스 위험

#### Risk 4: 시장 수용 저조
- **위험**: 대학들이 기존 도구 고수
- **영향**: 높음 - 매출 목표 미달
- **확률**: 중간
- **대응**:
  - 파일럿 프로그램 (무료 시범 운영)
  - 교수 트레이닝 세션
  - 성공 사례 적극 홍보

#### Risk 5: 경쟁사 모방
- **위험**: Zoom 등이 유사 기능 추가
- **영향**: 높음 - 차별점 상실
- **확률**: 중간 (1-2년 후)
- **대응**:
  - 지속적 혁신 (AI, 개인화)
  - 교육 도메인 전문성 강화
  - 커뮤니티 구축 (Lock-in)

### 13.3 운영 위험

#### Risk 6: 데이터 유출
- **위험**: 학생 개인정보 해킹
- **영향**: 매우 높음 - 법적 책임, 신뢰 손실
- **확률**: 낮음
- **대응**:
  - 보안 감사 정기 실시
  - 암호화 철저히 (전송, 저장)
  - 침해 대응 계획 수립

#### Risk 7: 핵심 인력 이탈
- **위험**: 주요 개발자 퇴사
- **영향**: 중간 - 개발 지연
- **확률**: 중간
- **대응**:
  - 코드 문서화 철저
  - Pair Programming
  - 지식 공유 문화
